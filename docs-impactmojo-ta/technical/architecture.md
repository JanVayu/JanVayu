# கட்டமைப்பு

ImpactMojo நிலையான HTML/CSS/JS தளத்தைப் பயன்படுத்துகிறது, எந்த உருவாக்க படியும் இல்லை, அங்கீகாரத்திற்கு Supabase மற்றும் ஹோஸ்டிங்கிற்கு Netlify ஆதரவளிக்கிறது. மெதுவான இணைப்புகள் மற்றும் பழைய சாதனங்களில் பயனர்களை ஆதரிக்க கட்டமைப்பு வேண்டுமென்றே இலகுவாக வைக்கப்பட்டுள்ளது.

## முக்கிய கூறுகள்

### முதன்மை தளம் (impactmojo.in)

- **ஹோஸ்டிங்:** Netlify (நிலையான வரிசைப்படுத்தல்)
- **முன்னணி:** Vanilla HTML/CSS/JS — React இல்லை, Vue இல்லை, உருவாக்க கருவிகள் இல்லை
- **முக்கிய ஸ்கிரிப்டுகள்:** `auth.js`, `router.js`, `premium.js`, `resource-launch.js`
- முதன்மை தளத்தில் **சேவையக-பக்க சூழல் மாறிகள் தேவையில்லை**

### அங்கீகாரம் & டோக்கன் சேவை

- **வழங்குநர்:** Supabase Auth (மின்னஞ்சல் + கடவுச்சொல்)
- **டோக்கன் உருவாக்கம்:** `mint-resource-token` Netlify Edge Function
- **டோக்கன் வகை:** JWT (HMAC-SHA256 கையொப்பமிடப்பட்டது)
- **டோக்கன் ஆயுள்:** 5 நிமிடங்கள்
- **உரிமைக்கோரல்கள்:** பயனர் ID, வள பெயர், சந்தா நிலை

### தரவுத்தளம் (Supabase PostgreSQL)

21+ அட்டவணைகள் உட்பட:

| அட்டவணை | நோக்கம் |
|-------|---------|
| `profiles` | சந்தா நிலை மற்றும் நிலையுடன் பயனர் கணக்குகள் |
| `organizations` | நிர்வாகி மற்றும் இருக்கை வரம்புகளுடன் குழு கணக்குகள் |
| `course_progress` | தொகுதி நிறைவு கண்காணிப்பு |
| `bookmarks` | பயனர் ஒன்றுக்கு சேமிக்கப்பட்ட வளங்கள் |
| `certificates` | சரிபார்க்கக்கூடிய நிறைவு பதிவுகள் |
| `payments` | கட்டண வரலாறு மற்றும் சந்தா மேலாண்மை |
| `cohorts` | திட்டமிடப்பட்ட குழு கற்றல் அமர்வுகள் |
| `notifications` | செயலி-உள் மற்றும் மின்னஞ்சல் எச்சரிக்கை வரிசை |
| `user_preferences` | தீம், மொழி, அறிவிப்பு அமைப்புகள் |

அனைத்து அட்டவணைகளும் **Row-Level Security** ஐப் பயன்படுத்துகின்றன — பயனர்கள் தங்கள் சொந்த பதிவுகளை மட்டுமே பார்க்கவும் மாற்றவும் முடியும்.

### பிரீமியம் கருவி தளங்கள்

ஒவ்வொரு பிரீமியம் கருவியும் சுயாதீன Netlify வரிசைப்படுத்தலாக இயங்குகிறது:

| கருவி | விவரம் |
|------|-------------|
| **RQ Builder Pro** | ஆராய்ச்சிக் கேள்வி வடிவமைப்பு பணிமேடை |
| **Qual Insights Lab** | AI-உதவி தரமான நேர்காணல் பகுப்பாய்வு |
| **Statistical Code Converter** | R, Stata, SPSS மற்றும் Python இடையே மாற்றுகிறது |
| **VaniScribe** | 10+ தென்னாசிய மொழிகளில் AI எழுத்துப்பெயர்ப்பு |
| **DevData Practice** | பயிற்சி தரவுத்தொகுப்புகள் மற்றும் காட்சிப்படுத்தல் செய்முறைகள் |
| **Viz Cookbook** | தரவு காட்சிப்படுத்தல் வார்ப்புரு நூலகம் |
| **DevEcon Toolkit** | ஊடாடும் வளர்ச்சி பொருளாதார பகுப்பாய்வு செயலிகள் |

**ஏன் தனி வரிசைப்படுத்தல்கள்?** ஒவ்வொரு கருவியையும் தனிமைப்படுத்துவது சுயாதீன வரிசைப்படுத்தல் மற்றும் மறுசெய்கையை அனுமதிக்கிறது. ஒரு கருவி செயலிழந்தால் மற்றவை அல்லது முதன்மை தளம் பாதிக்கப்படாது.

### அணுகல் ஓட்டம் (பிரீமியம் கருவிகள்)

```
1. பயனர் impactmojo.in இல் "Launch Tool" ஐ கிளிக் செய்கிறார்
2. கிளையன்ட் Supabase auth டோக்கனுடன் mint-resource-token (POST) ஐ அழைக்கிறது
3. Edge Function சந்தா நிலையை வள தேவைகளுக்கு எதிராக சரிபார்க்கிறது
4. அங்கீகரிக்கப்பட்டால்: JWT டோக்கன் + திருப்பி URL ஐ திருப்பி அனுப்புகிறது
5. பயனர் ?token= அளவுருவுடன் கருவி தளத்திற்கு திருப்பிவிடப்படுகிறார்
6. கருவியின் auth-gate Edge Function JWT கையொப்பத்தை சரிபார்க்கிறது
7. செல்லுபடியாகும் என்றால்: 24-மணி நேர resource_session குக்கீயை அமைக்கிறது
8. பயனர் மீண்டும் அங்கீகாரம் இல்லாமல் 24 மணி நேரம் கருவியை அணுகுகிறார்
```

## வடிவமைப்பு முடிவுகள்

| முடிவு | நியாயம் |
|----------|-----------|
| உருவாக்க படி இல்லை | எந்த சாதனத்திலும் வேலை செய்கிறது; பங்களிக்க Node.js தேவையில்லை |
| கட்டமைப்புகளுக்கு மேல் Vanilla JS | மெதுவான இணைப்புகளில் வேகமான ஏற்றம்; பராமரிக்க எளிதானது |
| தனி கருவி வரிசைப்படுத்தல்கள் | பிழை தனிமைப்படுத்தல்; சுயாதீன அளவிடுதல் மற்றும் மறுசெய்கை |
| தனிப்பயன் பின்தளத்திற்கு மேல் Supabase | நிர்வகிக்கப்பட்ட auth, RLS, நிகழ்நேர சந்தாக்கள் உடனடியாக |
| குறுகிய-ஆயுள் JWT-கள் (5 நிமிடம்) | டோக்கன் தவறான பயன்பாட்டிற்கான சாளரத்தை குறைக்கிறது; அமர்வு குக்கீ தொடர்ச்சியான அணுகலை கையாளுகிறது |
| கைமுறை UPI கட்டணங்கள் | இந்திய பயனர்களுக்கு கட்டண நுழைவாயில் கட்டணங்களை நீக்குகிறது |
