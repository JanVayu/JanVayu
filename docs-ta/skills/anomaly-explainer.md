# Skill: முரண்பாடு விளக்கி

**பயன்படுத்தப்படுவது:** `netlify/functions/anomaly-check.mjs`
**Model:** Gemini 2.5 Flash
**தூண்டுதல்:** தேவைக்கேற்ப அழைக்கப்படுகிறது; ஐந்து முக்கிய நகரங்களை பருவகால அடிப்படை மதிப்புகளுக்கு எதிராக சோதிக்கிறது

---

## இந்த Skill என்ன செய்கிறது

ஐந்து முக்கிய நகரங்களில் நேரடி அளவீடுகளை hardcode செய்யப்பட்ட பருவகால அடிப்படை மதிப்புகளுடன் ஒப்பிட்டு PM2.5 எகிற்சிகளை கண்டறிகிறது. எகிற்சி கண்டறியப்படும்போது (அளவீடு > அடிப்படை மதிப்பின் 2 மடங்கு), இந்தியா-குறிப்பிட்ட ஒரு வாக்கியத்தில் மிகவும் சாத்தியமான காரணத்தை விளக்க Gemini-யிடம் கேட்கிறது.

நான்கு skill files-ல் இது மிகவும் கட்டுப்படுத்தப்பட்டது — ஒரு வாக்கியம், ஒரு காரணம், அதிகபட்ச குறிப்பிட்டத்தன்மை.

---

## System Prompt (நகரம்-வாரி, துல்லியம்)

```
In one sentence, explain why PM2.5 in {city} might be {pm25} µg/m³ right now
— {month}, {hour}. Give the most likely cause. Be specific to the Indian context.
```

குறிப்பு: இது system instruction அல்ல, **user message**-ஆக அனுப்பப்படுகிறது. தெளிவான உண்மை அமைப்புடன் ஒரு வாக்கிய output-க்கு, user-turn prompt போதுமானது.

---

## எகிற்சி கண்டறிதல் Logic (AI-க்கு முன்)

```javascript
const ratio = d.pm25 / baseline;
if (ratio > 2) {
  // எகிற்சியாக flag செய், பிறகு Gemini வழியாக விளக்கு
}
```

வரம்பு: **பருவகால அடிப்படை மதிப்பின் 2 மடங்கு** (நிலையான absolute எண் அல்ல). இது முக்கியம் ஏனெனில்:
- டெல்லியின் குளிர்கால அடிப்படை ~95 µg/m³ — ஜனவரியில் 120 µg/m³ அளவீடு முரண்பாடு அல்ல
- மும்பையின் அடிப்படை ~45 µg/m³ — அங்கு 120 µg/m³ மிகவும் முரண்பாடு
- நிலையான வரம்பு (எ.கா., "150 µg/m³-க்கு மேல் உள்ள எதையும் flag செய்") false positives (டெல்லி குளிர்காலம்) மற்றும் false negatives (கடலோர நகரங்கள்) இரண்டையும் உருவாக்கும்

**பருவகால அடிப்படை மதிப்புகள் (CREA / IQAir historical data-லிருந்து):**

| நகரம் | அக்–மார் (குளிர்காலம்) | ஏப்–செப் (கோடை/மழைக்காலம்) |
|------|-----------------|--------------------------|
| டெல்லி | 95 µg/m³ | 55 µg/m³ |
| மும்பை | 45 µg/m³ | 45 µg/m³ |
| கொல்கத்தா | 80 µg/m³ | 35 µg/m³ |
| சென்னை | 40 µg/m³ | 40 µg/m³ |
| பெங்களூரு | 40 µg/m³ | 40 µg/m³ |

## முக்கிய வடிவமைப்பு முடிவுகள்

**ஏன் ஒரு வாக்கியம்?**
முரண்பாடு விளக்கம் UI-ல் tooltip அல்லது inline annotation-ஆக தோன்றுகிறது. பயனரின் flow-ஐ தடைப்படுத்தாமல் படிக்கக்கூடிய அதிகபட்சம் ஒரு வாக்கியம்.

**ஏன் prompt-ல் மாதம் மற்றும் மணி நேரத்தை சேர்க்கிறோம்?**
காரண பண்புக்கூறு நேரம் மற்றும் பருவத்தை பெரிதும் சார்ந்தது:
- நவம்பரில் காலை 7 மணிக்கு டெல்லி → பயிர் எரிப்பு + காலை inversion
- ஏப்ரலில் மதியம் 2 மணிக்கு டெல்லி → தூசி புயல்கள்
- நள்ளிரவில் மும்பை → தொழிற்சாலை/கப்பல் செயல்பாடு

**ஏன் Gemini அழைப்புகளை parallel-ல் (`Promise.allSettled`) இயக்குகிறோம்?**
ஒரே நேரத்தில் பல நகரங்களில் எகிற்சிகள் இருக்கலாம். `Promise.allSettled` (Promise.all அல்ல) ஒரு நகரத்தின் விளக்கம் தோல்வியடைந்தால் மற்றவை ரத்தாகாது.

## Fallback நடத்தை

```javascript
if (!s.explanation) {
  s.explanation = `PM2.5 is ${s.ratio}x above seasonal baseline.`;
}
```

Gemini கிடைக்காவிட்டால், பதிலில் spike data மற்றும் ratio இன்னும் உள்ளடங்கும் — AI-உருவாக்கப்பட்ட காரண வாக்கியம் இல்லாமல் மட்டுமே.
