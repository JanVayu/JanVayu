# Claude Code மேம்பாட்டு பணிப்பாய்வு

இந்தப் பக்கம் Claude Code-உடன் JanVayu-ஐ உருவாக்கும் அன்றாட பணிப்பாய்வை ஆவணப்படுத்துகிறது — அம்ச மேம்பாடு முதல் பிழை திருத்தங்கள் மற்றும் ஆவணங்கள் வரை.

---

## அம்ச மேம்பாட்டு பணிப்பாய்வு

### படி 1: அம்சத்தை வரையறுக்கவும்

தெளிவான, கட்டுப்பாடு-விழிப்புணர்வு விவரிப்புடன் தொடங்கவும்:

```
> வார்டு அளவிலான பொறுப்புணர்வு சுருக்க அம்சத்தை சேர்க்கவும்.
> இது ஒரு Netlify Function (ES module, .mjs) ஆக இருக்க வேண்டும்:
> - நகரப் பெயரை உள்ளீடாக எடுக்கும்
> - WAQI-லிருந்து நேரடி AQI-ஐ பெறும்
> - கட்டமைக்கப்பட்ட prompt-உடன் தரவை Gemini-க்கு அனுப்பும்
> - தற்போதைய நிலை, NCAP இலக்குகள், 5 பொறுப்புணர்வு கேள்விகளுடன் சுருக்கத்தை திருப்பும்
> - Gemini rate-limited ஆக இருந்தால் fallback கொண்டிருக்கும்
> ஏற்கனவே உள்ள patterns-ஐ பின்பற்றி index.html-ல் UI section-ஐயும் சேர்க்கவும்.
```

### படி 2: Claude ஆராய்கிறது

Claude Code படிக்கிறது:
- `index.html` — ஏற்கனவே உள்ள UI patterns, CSS variables, JS conventions
- ஏற்கனவே உள்ள Netlify Functions — code style, CORS handling, error patterns
- `package.json` — கிடைக்கும் dependencies
- `netlify.toml` — deployment configuration

### படி 3: Claude செயல்படுத்துகிறது

Claude கோப்புகளை உருவாக்குகிறது/மாற்றுகிறது:
1. `netlify/functions/accountability-brief.mjs` — serverless function
2. `index.html` — புதிய HTML section, CSS styles, JS fetch logic

### படி 4: மதிப்பாய்வு மற்றும் மீண்டும் செய்தல்

```
> சுருக்கம் மிகவும் நீளமாக உள்ளது. Gemini output-ஐ 400 tokens-க்கு கட்டுப்படுத்தவும்.
> சுருக்கம் உருவாகும்போது loading spinner-ஐயும் சேர்க்கவும்.
```

Claude தொடர்பில்லாத code-ஐ மாற்றாமல் குறிவைத்த மாற்றங்களை செய்கிறது.

### படி 5: Commit மற்றும் PR

```
> இதை "Add ward-level accountability brief feature" message-உடன் commit செய்து
> PR உருவாக்கவும்
```

Claude `git add`, `git commit`, `git push` மற்றும் `gh pr create` இயக்குகிறது.

## பிழை திருத்த பணிப்பாய்வு

```
> anomaly-check function WAQI down ஆகும்போது 500 திருப்புகிறது.
> cached data அடிப்படையில் anomaly flag-உடன் 200 திருப்ப வேண்டும்.
```

Claude Code:
1. `anomaly-check.mjs`-ஐ படிக்கிறது
2. கையாளப்படாத error path-ஐ கண்டறிகிறது
3. cached data-க்கு fallback-உடன் try/catch சேர்க்கிறது
4. fix-ஐ commit செய்கிறது

**முக்கிய கொள்கை:** Claude *குறைந்தபட்ச மாற்றத்தை* செய்ய அறிவுறுத்தப்படுகிறது — முழு function-ஐயும் refactor செய்யாது.

## நன்றாக வேலை செய்யும் Prompt Patterns

### புதிய அம்சங்களுக்கு

```
[அம்சத்தை] சேர்க்கவும். தேவைகள்:
- [கட்டுப்பாடு 1]
- [கட்டுப்பாடு 2]
[குறிப்பு கோப்பில்] உள்ள ஏற்கனவே உள்ள patterns-ஐ பின்பற்றவும்.
ஏற்கனவே உள்ள functionality-ஐ மாற்ற வேண்டாம்.
```

### பிழை திருத்தங்களுக்கு

```
[Function] [நிபந்தனையில்] [பிழையை] திருப்புகிறது.
எதிர்பார்க்கப்படுவது: [சரியான நடத்தை].
இதை சரிசெய்ய குறைந்தபட்ச மாற்றத்தை செய்யவும்.
சுற்றியுள்ள code-ஐ refactor செய்ய வேண்டாம்.
```

### ஆவணங்களுக்கு

```
[தலைப்புக்கான] ஆவணங்களை உருவாக்கவும்.
உள்ளடக்கம்: [குறிப்பிட்ட பிரிவுகள்].
Repo-லிருந்து உண்மையான code மற்றும் configuration-ஐ குறிப்பிடவும்.
[ஏற்கனவே உள்ள doc file] போன்ற தொனியை பயன்படுத்தவும்.
```

## இந்த பணிப்பாய்வை பயனுள்ளதாக்குவது

1. **முழு codebase சூழல்** — Claude code எழுதுவதற்கு முன் உங்கள் code-ஐ படிக்கிறது
2. **கட்டுப்பாடு-முதல் prompts** — Claude-க்கு என்ன செய்ய *கூடாது* என்று சொல்வது framework bloat-ஐ தடுக்கிறது
3. **படிப்படியான மெருகூட்டல்** — மதிப்பாய்வு, சரிசெய்தல், commit ஒரே session-ல்
4. **Git-native** — branches, commits, PRs ஒரே கருவிக்குள் நடக்கின்றன
5. **Multi-file ஒருங்கிணைப்பு** — ஒரு prompt functions, HTML, CSS, JS மற்றும் docs-ஐ தொடலாம்
6. **மறுஉருவாக்கக்கூடியது** — ஒவ்வொரு session-ன் வேலையும் தெளிவான messages-உடன் git-க்கு commit ஆகிறது
