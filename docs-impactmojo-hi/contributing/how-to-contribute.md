# योगदान कैसे करें

ImpactMojo ओपन सोर्स है। हम कोड, सामग्री, अनुवाद, और विचारों के योगदान का स्वागत करते हैं।

---

## योगदान के तरीके

| प्रकार | कैसे |
|------|-----|
| **बग रिपोर्ट** | [GitHub Issue](https://github.com/JanVayu/ImpactMojo/issues) खोलें |
| **फ़ीचर अनुरोध** | [GitHub Discussion](https://github.com/JanVayu/ImpactMojo/discussions) शुरू करें |
| **कोड योगदान** | Fork → branch → PR (नीचे कार्यप्रवाह देखें) |
| **सामग्री योगदान** | पाठ्यक्रम, केस स्टडी, हैंडआउट, DevDiscourses |
| **अनुवाद** | [अनुवाद](translations.md) देखें |
| **डेटा योगदान** | Dataverse के लिए डेटासेट |

---

## कोड योगदान कार्यप्रवाह

```bash
# 1. GitHub पर रिपो फ़ोर्क करें

# 2. अपना फ़ोर्क क्लोन करें
git clone https://github.com/<your-username>/ImpactMojo.git
cd ImpactMojo

# 3. एक ब्रांच बनाएँ
git checkout -b feature/your-feature-name

# 4. बदलाव करें (कोई बिल्ड चरण आवश्यक नहीं — सीधे HTML/CSS/JS संपादित करें)

# 5. स्थानीय रूप से परीक्षण करें
# ब्राउज़र में index.html खोलें, या उपयोग करें:
npx serve .

# 6. स्पष्ट संदेश के साथ कमिट करें
git commit -m "Add: description of your change"

# 7. पुश करें और PR खोलें
git push origin feature/your-feature-name
```

### कमिट संदेश परंपरा

| उपसर्ग | किसके लिए उपयोग |
|--------|---------|
| `Add` | नई सुविधा या सामग्री |
| `Fix` | बग फ़िक्स |
| `Update` | मौजूदा सुविधा में सुधार |
| `Translate` | अनुवाद जोड़ना या सुधार |
| `Docs` | प्रलेखन परिवर्तन |
| `Refactor` | व्यवहार परिवर्तन के बिना कोड पुनर्गठन |
| `Chore` | रखरखाव, निर्भरताएँ, CI |

## सामग्री योगदान

### पाठ्यक्रम
- फ़्लैगशिप या आधारभूत पाठ्यक्रम टेम्पलेट का पालन करें
- प्रमुख शब्दों की शब्दावली शामिल करें
- जहाँ प्रासंगिक हो दक्षिण एशियाई केस स्टडी जोड़ें
- सरल भाषा में लिखें — लक्ष्य पठन स्तर स्नातक है

### केस स्टडी
- किसी भी देश के वास्तविक कार्यक्रम
- देश, विषय, और कार्यप्रणाली के अनुसार टैग करें
- स्रोत और उद्धरण शामिल करें

### हैंडआउट
- प्रति पृष्ठ एक अवधारणा
- सरल भाषा, न्यूनतम शब्दजाल
- जहाँ संभव हो एक दृश्य (आरेख, तालिका, या फ़्लोचार्ट) शामिल करें
- प्राथमिक स्रोतों का हवाला दें

### Dataverse प्रविष्टियाँ
- शामिल करें: डेटासेट नाम, स्रोत, पहुँच विधि (डाउनलोड/API/MCP), टैग
- स्पष्ट लाइसेंसिंग के साथ ओपन-एक्सेस डेटासेट को प्राथमिकता दें
