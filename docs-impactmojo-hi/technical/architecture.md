# आर्किटेक्चर

ImpactMojo बिना किसी बिल्ड चरण के एक स्टैटिक HTML/CSS/JS साइट का उपयोग करता है, जो प्रमाणीकरण के लिए Supabase और होस्टिंग के लिए Netlify द्वारा समर्थित है। आर्किटेक्चर जानबूझकर हल्का है ताकि धीमे कनेक्शन और पुराने उपकरणों पर उपयोगकर्ताओं का समर्थन किया जा सके।

## मुख्य घटक

### मुख्य साइट (impactmojo.in)

- **होस्टिंग:** Netlify (स्टैटिक डिप्लॉयमेंट)
- **फ़्रंटएंड:** वैनिला HTML/CSS/JS — कोई React नहीं, कोई Vue नहीं, कोई बिल्ड उपकरण नहीं
- **प्रमुख स्क्रिप्ट:** `auth.js`, `router.js`, `premium.js`, `resource-launch.js`
- मुख्य साइट पर **कोई सर्वर-साइड एनवायरनमेंट वेरिएबल आवश्यक नहीं**

### प्रमाणीकरण और टोकन सेवा

- **प्रदाता:** Supabase Auth (ईमेल + पासवर्ड)
- **टोकन जनरेशन:** `mint-resource-token` Netlify Edge Function
- **टोकन प्रकार:** JWT (HMAC-SHA256 हस्ताक्षरित)
- **टोकन जीवनकाल:** 5 मिनट
- **दावे:** उपयोगकर्ता ID, संसाधन नाम, सदस्यता स्तर

### डेटाबेस (Supabase PostgreSQL)

21+ तालिकाएँ जिनमें शामिल हैं:

| तालिका | उद्देश्य |
|-------|---------|
| `profiles` | सदस्यता स्तर और स्थिति के साथ उपयोगकर्ता खाते |
| `organizations` | व्यवस्थापक और सीट सीमाओं के साथ टीम खाते |
| `course_progress` | मॉड्यूल पूर्णता ट्रैकिंग |
| `bookmarks` | प्रति उपयोगकर्ता सहेजे गए संसाधन |
| `certificates` | सत्यापन योग्य पूर्णता रिकॉर्ड |
| `payments` | भुगतान इतिहास और सदस्यता प्रबंधन |
| `cohorts` | अनुसूचित समूह शिक्षण सत्र |
| `notifications` | इन-ऐप और ईमेल अलर्ट कतार |
| `user_preferences` | थीम, भाषा, सूचना सेटिंग्स |

सभी तालिकाएँ **Row-Level Security** का उपयोग करती हैं — उपयोगकर्ता केवल अपने रिकॉर्ड देख और संशोधित कर सकते हैं।

### प्रीमियम उपकरण साइटें

प्रत्येक प्रीमियम उपकरण एक स्वतंत्र Netlify डिप्लॉयमेंट के रूप में चलता है:

| उपकरण | विवरण |
|------|-------------|
| **RQ Builder Pro** | शोध प्रश्न डिज़ाइन कार्यशाला |
| **Qual Insights Lab** | AI-सहायित गुणात्मक साक्षात्कार विश्लेषण |
| **Statistical Code Converter** | R, Stata, SPSS, और Python के बीच रूपांतरण |
| **VaniScribe** | 10+ दक्षिण एशियाई भाषाओं में AI ट्रांसक्रिप्शन |
| **DevData Practice** | अभ्यास डेटासेट और विज़ुअलाइज़ेशन रेसिपी |
| **Viz Cookbook** | डेटा विज़ुअलाइज़ेशन टेम्पलेट लाइब्रेरी |
| **DevEcon Toolkit** | इंटरैक्टिव विकास अर्थशास्त्र विश्लेषण ऐप्स |

**अलग डिप्लॉयमेंट क्यों?** प्रत्येक उपकरण को अलग करने से स्वतंत्र डिप्लॉयमेंट और पुनरावृत्ति संभव होती है। एक उपकरण के बंद होने से दूसरे या मुख्य साइट प्रभावित नहीं होते।

### पहुँच प्रवाह (प्रीमियम उपकरण)

```
1. उपयोगकर्ता impactmojo.in पर "Launch Tool" पर क्लिक करता है
2. क्लाइंट Supabase auth टोकन के साथ mint-resource-token (POST) कॉल करता है
3. Edge Function संसाधन आवश्यकताओं के विरुद्ध सदस्यता स्तर की जाँच करता है
4. यदि अधिकृत: JWT टोकन + रीडायरेक्ट URL लौटाता है
5. उपयोगकर्ता को ?token= पैरामीटर के साथ उपकरण साइट पर रीडायरेक्ट किया जाता है
6. उपकरण का auth-gate Edge Function JWT हस्ताक्षर सत्यापित करता है
7. यदि मान्य: 24-घंटे की resource_session कुकी सेट करता है
8. उपयोगकर्ता 24 घंटे तक बिना पुनः प्रमाणीकरण के उपकरण का उपयोग करता है
```

## डिज़ाइन निर्णय

| निर्णय | तर्क |
|----------|-----------|
| कोई बिल्ड चरण नहीं | किसी भी उपकरण पर काम करता है; योगदान के लिए Node.js की आवश्यकता नहीं |
| फ़्रेमवर्क के बजाय वैनिला JS | धीमे कनेक्शन पर तेज़ लोड; रखरखाव आसान |
| अलग उपकरण डिप्लॉयमेंट | दोष अलगाव; स्वतंत्र स्केलिंग और पुनरावृत्ति |
| कस्टम बैकएंड के बजाय Supabase | प्रबंधित auth, RLS, रीयल-टाइम सदस्यताएँ तैयार |
| अल्पकालिक JWT (5 मिनट) | टोकन दुरुपयोग की संभावना कम करता है; सत्र कुकी चालू पहुँच संभालती है |
| मैनुअल UPI भुगतान | भारतीय उपयोगकर्ताओं के लिए भुगतान गेटवे शुल्क समाप्त करता है |
