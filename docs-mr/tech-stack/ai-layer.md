# AI स्तर — Google Gemini 2.5 Flash

JanVayu ची AI वैशिष्ट्ये **Google Gemini 2.5 Flash** मॉडेलद्वारे चालतात, Netlify Functions मधून `@google/generative-ai` npm package द्वारे ऍक्सेस केले जातात.

---

## Gemini 2.5 Flash का?

| निकष | निवड |
|-----------|--------|
| **किंमत** | फ्री टियर (Google AI Studio) — बिलिंग आवश्यक नाही |
| **वेग** | कमी-विलंब प्रतिसादांसाठी ऑप्टिमाइझ केलेला Flash variant |
| **रेट लिमिट्स** | 250 विनंत्या/दिवस, 10/मिनिट — सार्वजनिक हिताच्या व्यासपीठासाठी पुरेसे |
| **बहुभाषिक** | मजबूत हिंदी समर्थन (JanVayu च्या प्रेक्षकांसाठी गंभीर) |
| **गुणवत्ता** | डेटा-आधारित तथ्यात्मक प्रतिसादांसाठी पुरेसे (सर्जनशील लेखनासाठी नाही) |

### स्वीकारलेले तडजोड

- **GPT-4 / Claude नाही** — Gemini चा फ्री टियर हा फरक आहे. JanVayu शून्य बजेटवर चालतो.
- **Output token cap** — सर्व प्रॉम्प्ट आउटपुट 150-400 tokens पर्यंत मर्यादित करतात. हे वैशिष्ट्य आहे: संक्षिप्त प्रतिसाद नागरी डेटासाठी अधिक उपयुक्त आहेत.
- **Fine-tuning नाही** — फक्त prompt engineering. प्रत्येक skill ही system prompt आहे, fine-tuned model नाही.

## Integration आर्किटेक्चर

```
Browser (client)
    │
    │ POST /.netlify/functions/air-query
    │ POST /.netlify/functions/health-advisory
    │ POST /.netlify/functions/accountability-brief
    │ GET  /.netlify/functions/anomaly-check
    │
    ▼
Netlify Function (server-side)
    │
    │ 1. वापरकर्ता इनपुट parse करा
    │ 2. WAQI वरून लाइव्ह AQI डेटा आणा
    │ 3. वास्तविक डेटासह prompt तयार करा
    │ 4. Gemini API कॉल करा
    │ 5. प्रतिसाद परत करा (किंवा raw data वर fallback)
    │
    ▼
Google Gemini API
    │
    │ gemini-2.5-flash model
    │ maxOutputTokens: 150-400
    │ temperature: 0.3-0.7
    │
    ▼
संरचित प्रतिसाद → Browser
```

**प्रमुख सुरक्षा तत्त्व:** `GEMINI_API_KEY` कधीही client ला स्पर्श करत नाही. सर्व AI कॉल Netlify Functions द्वारे सर्व्हर-साइड आहेत.

## चार AI वैशिष्ट्ये

### 1. Ask JanVayu (`air-query.mjs`)
- **इनपुट:** शहराचे नाव + फ्री-टेक्स्ट प्रश्न
- **इंजेक्ट केलेला संदर्भ:** WAQI वरून लाइव्ह PM2.5 आणि AQI
- **आउटपुट:** < 150 शब्द, वास्तविक रीडिंगवर आधारित
- **भाषा:** English आणि Hindi
- **Fallback:** रेट-लिमिटेड असल्यास raw AQI डेटा परत करतो

### 2. आरोग्य सल्ला (`health-advisory.mjs`)
- **इनपुट:** शहर + वय + आरोग्य स्थिती + बाहेरचे तास
- **इंजेक्ट केलेला संदर्भ:** त्या शहरासाठी लाइव्ह AQI
- **आउटपुट:** रंगकोडित जोखीम पातळी + ठोस शिफारसी
- **Fallback:** सामान्य WHO-मार्गदर्शक सल्ला

### 3. उत्तरदायित्व ब्रीफ (`accountability-brief.mjs`)
- **इनपुट:** शहराचे नाव
- **इंजेक्ट केलेला संदर्भ:** लाइव्ह AQI + हंगामी baselines + GRAP stages
- **आउटपुट:** संरचित ब्रीफ (सध्याची स्थिती, NCAP लक्ष्ये, 5 उत्तरदायित्व प्रश्न)
- **Max tokens:** 400
- **Fallback:** raw data table परत करतो

### 4. विसंगती शोध (`anomaly-check.mjs`)
- **इनपुट:** काहीही नाही (5 metros स्वयंचलितपणे मॉनिटर करतो)
- **Threshold:** 2× हंगामी baseline = विसंगती
- **आउटपुट:** प्रति विसंगती एक-वाक्य AI स्पष्टीकरण
- **कॅश:** 10-मिनिट Netlify Blobs कॅश
- **Fallback:** AI स्पष्टीकरणाशिवाय विसंगती flag परत करतो

## रेट लिमिटिंग धोरण

फ्री-टियर Gemini मर्यादा:

| मर्यादा | मूल्य |
|-------|-------|
| दररोज विनंत्या | 250 |
| प्रति मिनिट विनंत्या | 10 |
| प्रति मिनिट tokens | 1,000,000 |

JanVayu हे बजेट वैशिष्ट्यांमध्ये वितरित करतो:
- Anomaly check: ~144/दिवस (कॅश केलेले, पेज लोडवर fire होते)
- Air query: ~50/दिवस (वापरकर्ता-सुरू)
- Health advisory: ~30/दिवस (वापरकर्ता-सुरू)
- Accountability brief: ~20/दिवस (वापरकर्ता-सुरू)

मर्यादा गाठली गेल्यास, प्रत्येक function सुंदरपणे raw data वर fallback होतो. वापरकर्त्याला नेहमी प्रतिसाद मिळतो — फक्त AI विश्लेषणाशिवाय.

## Prompt Engineering

सर्व Gemini prompts या तत्त्वांचे पालन करतात:

1. **नेहमी वास्तविक डेटा इंजेक्ट करा** — मॉडेल कधीही स्मृतीतून तयार करत नाही
2. **अचूक output format निर्दिष्ट करा** — विशेषतः संरचित ब्रीफसाठी
3. **शब्द संख्या मर्यादित करा** — प्रत्येक prompt मध्ये स्पष्ट मर्यादा
4. **अयशस्वी modes नाव द्या** — "सामान्य सल्ला देऊ नका", "टाळू नका"
5. **बहुभाषिक सूचना** — "प्रश्नाच्या भाषेत प्रतिसाद द्या"
6. **सुंदर degradation** — प्रत्येक function कडे नॉन-AI fallback path आहे

पूर्ण prompt documentation साठी [कौशल्ये आणि AI प्रॉम्प्ट्स](../skills/README.md) विभाग पहा.
