# How to Contribute

ImpactMojo is open source. We welcome contributions of code, content, translations, and ideas.

---

## Ways to Contribute

| Type | How |
|------|-----|
| **Bug reports** | Open a [GitHub Issue](https://github.com/JanVayu/ImpactMojo/issues) |
| **Feature requests** | Start a [GitHub Discussion](https://github.com/JanVayu/ImpactMojo/discussions) |
| **Code contributions** | Fork → branch → PR (see workflow below) |
| **Content contributions** | Courses, case studies, handouts, DevDiscourses |
| **Translations** | See [Translations](translations.md) |
| **Data contributions** | Datasets for the Dataverse |

---

## Code Contribution Workflow

```bash
# 1. Fork the repo on GitHub

# 2. Clone your fork
git clone https://github.com/<your-username>/ImpactMojo.git
cd ImpactMojo

# 3. Create a branch
git checkout -b feature/your-feature-name

# 4. Make changes (no build step needed — edit HTML/CSS/JS directly)

# 5. Test locally
# Open index.html in a browser, or use:
npx serve .

# 6. Commit with a clear message
git commit -m "Add: description of your change"

# 7. Push and open a PR
git push origin feature/your-feature-name
```

### Commit Message Convention

| Prefix | Use For |
|--------|---------|
| `Add` | New feature or content |
| `Fix` | Bug fix |
| `Update` | Enhancement to existing feature |
| `Translate` | Translation additions or improvements |
| `Docs` | Documentation changes |
| `Refactor` | Code restructuring without behaviour change |
| `Chore` | Maintenance, dependencies, CI |

## Content Contributions

### Courses
- Follow the flagship or foundational course template
- Include a lexicon of key terms
- Add South Asian case studies where relevant
- Write in plain language — target reading level is undergraduate

### Case Studies
- Real-world programmes from any country
- Tag by country, theme, and methodology
- Include sources and citations

### Handouts
- One concept per page
- Plain language, minimal jargon
- Include a visual (diagram, table, or flowchart) where possible
- Cite primary sources

### Dataverse Entries
- Include: dataset name, source, access method (download/API/MCP), tags
- Prefer open-access datasets with clear licensing
